Part II

A First Dip into the Pond of HoTT

The Challenge: H-Levels

In HoTT, types have homotopy structure:

  • Contractible — exactly one element
  • Proposition — at most one element
  • Set — paths between elements are unique
  • Groupoid — paths can differ...

Classical realizability implicitly assumes sets.

In HoTT, we must be explicit.

Conservative Approach

record HoTT-EvidencedFrame : Type where
  field
    Φ : Type
    Φ-isSet : isSet Φ      -- Explicit!

    E : Type
    E-isSet : isSet E      -- Explicit!

    _⊢[_]_ : Φ → E → Φ → Type

Stay close to classical, but gain HoTT tools

An Observation

Uniformity ↔ Transport

Classical:

"Same evidence works for all states"

HoTT:

"Evidence transports along paths"

This reframing appears to be novel — I'd value your perspective.

What is Transport?

If you have a path p : σ₁ ≡ σ₂ and a realizer in σ₁...

Transport carries the realizer along the path to σ₂:

transport (λ σ → ⟦ φ ⟧ σ c) p realizer₁ : ⟦ φ ⟧ σ₂ c

The path p is like a "movie" of σ₁ morphing into σ₂.
The realizer "rides along" as the state continuously changes.

Uniformity = continuous dependence on parameters

Classical: "e works for σ₁" and "e works for σ₂" are separate facts.
HoTT: If σ₁ ≡ σ₂, these facts are connected — one transports to the other.

The Formal Statement

-- Given: evidence e, a path between states, a realizer at σ₁
-- Result: a realizer at σ₂

evidence-respects-paths :
    φ₁ ⊢[ e ] φ₂        -- e is evidence (unused here!)
  → (p : σ₁ ≡ σ₂)       -- states are equal via path p
  → ⟦ φ₁ ⟧ σ₁ c         -- realizer at σ₁
  → ⟦ φ₁ ⟧ σ₂ c         -- realizer at σ₂

evidence-respects-paths _ p r = transport (λ i → ⟦ φ₁ ⟧ (p i) c) r

Uniformity gains geometric meaning:

"Same evidence works for all states" becomes
"realizers live in a continuous family over the space of states"

Part III

Research Directions

What's uniquely possible in HoTT

1. Path-Based Evidence

Classical: Evidence e : E is a code

HoTT: Evidence could be a path!

-- If p : φ₁ ≡ φ₂, then p IS evidence!
transport-realizer : (p : φ₁ ≡ φ₂) → ⟦ φ₁ ⟧ σ c → ⟦ φ₂ ⟧ σ c

-- Transitivity = path composition (automatic!)
compose-evidence : φ₁ ≡ φ₂ → φ₂ ≡ φ₃ → φ₁ ≡ φ₃
compose-evidence p q = p ∙ q
Question: Can we embed code-based evidence into path-based?

2. Univalence + Uniformity

Univalence: (A ≃ B) ≃ (A ≡ B)

Equivalent types are equal.

For evidenced frames:

  • If propositions are equivalent...
  • Univalence gives a path...
  • That path is evidence!
Question: Does univalence generate evidence automatically?

3. HITs as Propositions

What if Φ has higher structure?

Example: The Circle S¹

  • base : S¹
  • loop : base ≡ base (non-trivial!)

Propositions could "rotate" around the loop...

Question: What does "realize a point on S¹" mean?

4. HITs as Effects

HIT Effect Meaning
∥A∥ truncation Failure Knows something exists, forgot which one
Like fail: terminates but no value returned
Quotient A/~ Equality oracle Can test if a ~ b at runtime
Extensional equality as computational effect
Circle S¹ Periodicity State that cycles back to start
Modular arithmetic, cyclic processes
Question: Can HITs model effects that PCAs cannot?

5. Categorical Structure

Evidence as a type family:

Evidence : Φ → Φ → Type

This is categorical:

  • Objects = propositions
  • Morphisms = evidence

In HoTT: need coherence (associativity, units...)

Leads to ∞-categorical realizability

6. Synthetic Homotopy Realizability

Grand vision:

Synthetic homotopy theory defines spheres, suspensions
without topological spaces.

Could we define realizability
without specific PCAs?

  • Realizers for homotopy groups?
  • Evidence for path spaces?
  • Computational meaning for πₙ?
Completely novel territory — no existing work!
